# Модель угроз — Arx Notes `:crypto`

Документ фиксирует, **от чего защищает** крипто-ядро, **от чего нет**, и какие
допущения сделаны. Это часть «честности» открытого кода: без явных границ любая
криптография — лишь видимость защиты.

## Что защищаем (активы)
- Содержимое заметок и задач (в зашифрованной БД SQLCipher).
- Аудио голосовых заметок (зашифрованные файлы).
- PIN блокировки — хранится как односторонний Argon2id-хеш (не в восстановимом виде).
- Файлы бэкапа, выгруженные наружу через SAF.

## Защищённые сценарии
| Угроза | Защита |
|---|---|
| Доступ к файлам приложения без разблокировки устройства (потеря/кража выключенного/заблокированного телефона, извлечение чипа памяти) | БД/аудио зашифрованы ключом, производным от мастера, обёрнутого ключом **Android Keystore** (аппаратно, где даёт платформа — см. допущения); ключ-обёртка не извлекается. |
| Чтение файла БД сторонним SQLite-клиентом | SQLCipher: файл нечитаем без ключа. |
| Кража/перехват файла бэкапа | Argon2id (память-жёсткий KDF) + AES-256-GCM; перебор пароля дорог; подмена → провал GCM. |
| Подмена/порча зашифрованных данных | Аутентифицированное шифрование (GCM-тег) везде; модификация → расшифровка возвращает ошибку/`null`. |
| Утечка одного рабочего подключа | HKDF-разделение: не раскрывает мастер и другие подключи. |

## Вне модели (НЕ защищаем)
- **Скомпрометированное/рутованное устройство, активная малварь с правами процесса** — может прочитать ключи в памяти работающего приложения.
- **Разблокированное устройство в руках атакующего** при выключенной блокировке входа (блокировка опциональна).
- **Слабый пароль бэкапа** — Argon2id повышает стоимость перебора, но не спасает тривиальные пароли.
- **Криминалистический дамп RAM** работающего процесса (ключи и расшифрованные данные неизбежно присутствуют в памяти во время работы).
- **Метаданные**: размеры файлов, время изменения, факт наличия заметок/аудио.
- Сторона UI/ОС: скриншоты, буфер обмена, клавиатурные логгеры, бэкап ОС — вне ответственности крипто-ядра.
- **Текст напоминания вне зашифрованного хранилища**: чтобы ОС показала уведомление на
  заблокированном экране, текст задачи кладётся в `PendingIntent` системного AlarmManager
  (переживает перезагрузку). Это вне крипто-ядра и неизбежно для работы уведомлений.

## Ключевые допущения
- Android Keystore честно защищает ключ-обёртку (TEE/StrongBox, где доступно; иначе — программно).
  Код не запрашивает StrongBox явно и не проверяет фактический уровень привязки — полагается на дефолт платформы.
- `SecureRandom` платформы даёт криптостойкую энтропию.
- Пароль бэкапа известен только пользователю и не хранится приложением.

## Обращение с ключами в памяти
- Транзитные ключевые буферы затираются после использования (`fill(0)`): мастер-секрет
  после вывода подключей; производный ключ бэкапа и копия пароля после Argon2.
- Долгоживущие рабочие ключи (парольная фраза SQLCipher, AEAD-ключи) **неизбежно**
  присутствуют в памяти, пока приложение работает, — защита от дампа RAM вне модели.
- Keystore-ключ-обёртка требует разблокированного устройства
  (`setUnlockedDeviceRequired(true)`, API 28+): на заблокированном (но включённом) телефоне
  ключ БД/аудио недоступен — защита от извлечения данных с потерянного заблокированного
  устройства. Функциональность не страдает: напоминания берут текст из интента (не из БД),
  а виджет под замком показывает «замок». На API < 28 не поддерживается (no-op); на устройстве
  без защищённого экрана блокировки — фактически no-op.

## Параметры (по умолчанию)
- Мастер-секрет: 256 бит, `SecureRandom`.
- KDF БД: SQLCipher (PBKDF2 поверх HKDF-подключа).
- KDF бэкапа: **Argon2id**, m=64 MiB, t=3, p=1 (параметры пишутся в заголовок файла).
- Хеш PIN: **Argon2id**, m=32 MiB, t=2, p=1, соль 16 байт; сравнение за постоянное время
  (параметры умереннее бэкапа: PIN на «горячем» пути разблокировки, но выше floor OWASP).
- Симметрика: AES-256-GCM, 96-бит nonce, 128-бит тег.
- HKDF: HMAC-SHA256, доменные `info`-метки.
